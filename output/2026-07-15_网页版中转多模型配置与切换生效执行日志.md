# 网页版中转多模型配置与切换生效执行日志

日期：2026-07-15
状态：已上线并完成线上核验

## 目标

- 同一 API 配置仅修改默认模型时，可以保存并在下一次模块请求中使用新模型。
- 对同一中转站 Key 可从现有配置派生多个“连接相同、模型独立”的 API 配置，供模块模型分配分别使用。
- 连接中心保持每个配置只配置一个模型，避免一个配置内部多选模型导致路由歧义。

## 已知根因

- 当前修改模型会重置测试状态；保存逻辑要求测试状态为 `ok`，因此未重测时保存失败、旧档案继续被请求层读取。
- OpenAI 兼容协议的“测试连接”只访问 `/models`，验证连接而非所选模型。
- 请求层不缓存模型：保存后的配置会被每次模块请求重新解析。

## 实施边界

- 只修改 Pages 仓的 `source/moyun.single.html` 与构建产物。
- 不修改冻结离线单文件、白鸟 JSON、历史交付与既有未跟踪测试文件。
- 模型切换不应自动更换地址、Key 或协议；用户需通过派生配置/模块分配显式切换连接线路。
- 单个模块的独立配置失败时仍直接显示上游错误，不使用其他配置兜底。

## 计划

1. 将连接验证状态拆分为“连接已验证”与“当前模型已验证”；模型单独修改允许保存，连接字段修改仍要求重测。
2. 为编辑中的模型添加最小真实模型验证；OpenAI 兼容协议保留模型发现，但模型验证走当前模型的 completion 请求。
3. 增加紧凑的“派生模型配置”入口，复制协议、地址、本机凭据与连接验证状态，仅要求填写/选择新模型。
4. 多样本抓包覆盖：未保存旧模型、同配置已保存新模型、同 Key 派生双配置、模块跨配置切换、不同协议错误直通；完成桌面/手机 UI 回归和 Pages 发布验证。
