# 首开新手说明爱发电跳转入口执行日志

日期：2026-07-15
状态：测试通过，待提交发布

## 目标

- 仅在网页版首开“新手说明”弹窗的最下方增加清晰的爱发电支持段落与“【前往爱发电】”跳转按钮，解决新用户找不到入口的反馈。
- 跳转地址固定为 `https://www.ifdian.net/a/moyunapp`，在新标签打开并使用 `noopener noreferrer`。
- 不改现有网页更新公告、关于页、连接中心、MOD、离线冻结区或其他运行逻辑。

## 实施计划

1. 为首开说明数据增加仅该公告使用的支持链接对象。
2. 在通用公告弹窗内容区的 sections 之后按条件渲染支持段落；更新公告没有该对象时保持原样。
3. 构建、检查桌面和手机视口下的新用户首开弹窗，确认链接、位置、可点击性和关闭逻辑。
4. 仅提交本次源文件、生成物和本日志，推送 `main` 后核验线上。

## 已完成

- 为 `MOYUN_FIRST_RUN_GUIDE` 增加 `supportLink`，包含标题、说明、致谢、`【前往爱发电】` 按钮文案与官方地址。
- 通用公告弹窗仅在当前公告拥有 `supportLink` 时，于所有说明 section 之后渲染支持段落；普通网页更新公告没有该字段，因此保持原样。
- 链接使用 `target="_blank"` 与 `rel="noopener noreferrer"`，按钮采用全宽、最小 44px 高度，便于手机触达。

## 校验与实测

- `npm run build` 通过；JS 语法检查和 `git diff --check` 通过。
- 使用本地静态站点与 Edge Playwright 实测新用户首开路径：桌面 `1440×900`、手机 `390×844` 均能滚至底部看到支持段落和按钮，无控制台错误。
- 点击按钮后真实新开 `https://www.ifdian.net/a/moyunapp`，页面标题为 Moyun 的爱发电主页。
- 点击“开始使用”并刷新后，按既有逻辑显示普通更新公告；其页面不含“请作者喝杯咖啡。”段落，确认没有误影响既有公告。
